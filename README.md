# CardioApp

Мобильное приложение для мониторинга здоровья детей с кардиологическими заболеваниями. Позволяет фиксировать показатели здоровья, отслеживать приём витаминов и медикаментов, получать напоминания и связываться с врачом.

## Функциональность

- **Сегодня** — карточка с текущими показателями здоровья и витаминами на день
- **Запись показателей** — ввод ЧСС, АД, SpO2, ЧД, МНО (INR) с цветовой индикацией нормы
- **История** — календарь записей с цветными точками по зонам, стрик и история витаминов
- **Профиль** — возрастная группа, медикаменты, витамины, кастомные лимиты показателей, настройки уведомлений
- **Помощь** — контакты врача, ссылка на опрос с индикатором нового опроса
- **Уведомления** — ежедневное напоминание в 12:00 и уведомление о новом опросе
- **Онбординг** — первичная настройка профиля пациента

## Архитектура

Multi-module Clean Architecture на Jetpack Compose.

```
app/                        # Application, DI-граф, навигация, воркеры
core/
  model/                    # Доменные модели (data class, enum)
  domain/                   # Интерфейсы репозиториев, use case
  data/                     # Реализации репозиториев, Room, DataStore, sync
  database/                 # Room БД, DAO, entity, seed data
  ui/                       # Дизайн-система, общие Compose компоненты
  common/                   # Утилиты
feature/
  today/                    # Экран "Сегодня"
  healthrecords/            # Создание, просмотр, редактирование записей
  history/                  # История записей
  profile/                  # Профиль и настройки
  help/                     # Помощь, контакты, опрос
  vitamins/                 # Управление витаминами
  onboarding/               # Онбординг
```

**Стек:** Kotlin · Jetpack Compose · Hilt · Room · DataStore · WorkManager · Navigation Compose · Yandex AppMetrica

## Настройка

### 1. API-ключ Яндекс AppMetrica

Создать файл `local.properties` в корне проекта (или добавить строку если файл уже есть):

```properties
appmetrica.api.key=ВАШ_КЛЮЧ
```

Получить ключ: [appmetrica.yandex.ru](https://appmetrica.yandex.ru) → приложение → Настройки → API key.

### 2. Remote Config (контакты и ссылка на опрос)

Данные о контактах врача, ссылке на опрос и его идентификаторе хранятся в JSON-файле на GitLab/GitHub. При запуске приложение загружает этот файл и обновляет локальную БД.

URL файла задан в `AppConfigSyncer.kt`:

```kotlin
private const val CONFIG_URL = "https://..."
```

Формат файла:

```json
{
  "survey_url": "https://ссылка-на-опрос",
  "survey_id": "survey_001",
  "contacts": [
    {
      "id": "contact-doctor",
      "title": "Доктор",
      "phone": "+7XXXXXXXXXX",
      "sort_order": 0
    },
    {
      "id": "contact-appointment",
      "title": "Запись на приём",
      "phone": "+7XXXXXXXXXX",
      "sort_order": 1
    }
  ]
}
```

Чтобы обновить контакты или ссылку на опрос — отредактировать JSON-файл. При следующем запуске приложения данные обновятся автоматически. При изменении `survey_id` пользователи увидят зелёную точку-индикатор нового опроса.

## Уведомления

- **Ежедневное напоминание** — приходит в 12:00 по местному времени устройства. Управляется из раздела Профиль.
- **Новый опрос** — приходит когда приложение в фоне обнаруживает новый `survey_id` в конфиге (проверка раз в 6 часов). При тапе открывает экран Помощь.

## Цветовые зоны показателей

| Цвет | Значение |
|---|---|
| Зелёный | Норма |
| Жёлтый | Требует внимания |
| Красный | Срочно к врачу |

Лимиты берутся из возрастной группы пациента или из кастомных значений, заданных в профиле.

## Аналитика (Яндекс AppMetrica)

| Событие | Когда |
|---|---|
| `app_opened` | Каждый запуск приложения |
| `first_open` | Первый запуск после установки |
| `measurement_added` | Добавлена запись показателей |
| `medicine_marked` | Отмечен приём медикамента |
| `questionnaire_opened` | Открыта ссылка на опрос |
| `questionnaire_completed` | Опрос завершён |
| `abnormal_value_detected` | Введено значение вне нормы |
| `call_button_pressed` | Нажата кнопка звонка врачу |
| `opened_from_daily_notification` | Переход в приложение по ежедневному уведомлению |
| `opened_from_survey_notification` | Переход в приложение по уведомлению о новом опросе |
